# ТЗ: Vanish-FFConverter

*Статус документа: в работе. Разделы добавляются по мере утверждения.*

## 1. Введение

### 1.1. Что за программа

**Vanish-FFConverter** — портативная Windows-программа, графическая оболочка
для ffmpeg, закрывающая повседневные задачи работы с аудио и видео: сжатие,
смена формата, извлечение и удаление дорожек, резка, склейка и инспекция
файлов. Один exe + папка ffmpeg, без установки и зависимостей.

Имя исполняемого файла: `Vanish-FFConverter.exe`.

### 1.2. Для кого и зачем

Для обычного пользователя, которому нужен результат, а не изучение командной
строки ffmpeg. Типовые сценарии:

- сжать запись встречи/подкаста в компактный файл для архива или пересылки;
- вытащить звук из видео;
- перегнать mkv в mp4 для телевизора/телефона;
- разрезать длинную запись на части, склеить диктофонные куски;
- быстро посмотреть, что внутри файла (кодеки, дорожки, битрейт).

### 1.3. Главные принципы

1. **Простота важнее полноты.** Каждая вкладка отвечает на один вопрос
   «зачем я открыл программу». Разумные значения по умолчанию: нажать
   «Старт» можно сразу, ничего не настраивая.
2. **Пресеты + расширенный режим.** Новичок выбирает пресет словами
   («Компактно — голос»), опытный раскрывает расширенный режим и крутит
   настройки кодека вручную.
3. **Формы и пачки, не таймлайн.** Программа работает списками файлов и
   параметрами. Никакого предпросмотра, плеера и визуальной резки — для
   этого есть Machete (рекомендация в README). Быстрый просмотр файла —
   кнопкой ffplay.
4. **Честность операций.** Мгновенные операции без потерь (`-c copy`) явно
   отделены от перекодирования. Где есть компромисс (рез по ключевым
   кадрам, несовместимость кодеков с контейнером) — программа говорит об
   этом прямо, а не молча портит результат.
5. **Исходники неприкосновенны.** Программа никогда не изменяет и не
   перезаписывает входные файлы; результат — всегда новый файл рядом или
   в выбранной папке.
6. **Тёмная и светлая тема**, переключение кнопкой на лету, без
   перезапуска.

### 1.4. Технологии и ограничения

- **Язык/GUI:** C# + WinForms, .NET Framework 4.8 (предустановлен в
  Windows 10/11). Сборка встроенным `csc` — без сторонних SDK.
- **Движок:** ffmpeg / ffprobe / ffplay в подпапке `ffmpeg` (программа —
  только оболочка; сама медиа не декодирует).
- **ОС:** Windows 10/11, 64-bit.
- **Интерфейс:** русский; все строки — в едином словаре для последующего
  добавления английского языка как опции.
- **Перетаскивание** файлов мышью в окно — во всех вкладках.
- **Настройки:** `settings.json` рядом с exe; сохраняются все параметры
  вкладок, тема, геометрия окна, последняя папка вывода. Запись — при
  выходе и перед стартом каждой операции. Реестр и AppData не
  используются.
- **Темы:** палитры «тёмная» и «светлая», прокраска всех элементов окна +
  тёмный заголовок окна (флаг DWM в Windows 10/11). Оговорка первой
  версии: выпадающие списки комбобоксов и полосы прокрутки остаются
  системного цвета (кастомная отрисовка — в поздние итерации).
- Результат сборки: один exe (сотни КБ) + папка `ffmpeg`
  (~80–280 МБ в зависимости от сборки ffmpeg).

### 1.5. Чего в программе нет (сознательно)

- Видеоредактирования: таймлайна, предпросмотра, фильтров, цветокоррекции,
  водяных знаков.
- Интерактивной визуальной резки «глазами» — эту нишу закрывает Machete.
- Записи экрана, стриминга, скачивания из интернета.
- Установщика, реестра, телеметрии, записи вне своей папки —
  портативность.

## 2. Структура интерфейса

### 2.1. Главное окно

- Одно окно с вкладками. Размер и позиция запоминаются; окно можно
  растягивать, минимальный размер ~900×600.
- Справа сверху, над вкладками: кнопка **переключения темы** (🌙/☀);
  в будущем рядом — переключатель языка.
- Внизу окна — **статусная полоса**, общая для всех вкладок:
  `В работе: запись.mp3 (Аудио) · в очереди: 4 · ⏸ пауза после текущего`.
  С любой вкладки видно, что делает программа.
- Заголовок окна: `Vanish-FFConverter`.

### 2.2. Вкладки

| № | Вкладка | Назначение | Тип операции |
|---|---------|-----------|--------------|
| 1 | **Аудио** | сжатие/конвертация звука: opus, mp3, aac, flac, wav; нормализация; сведение дорожек; каналы | перекодирование |
| 2 | **Видео** | сжатие видео: H.264/H.265, CRF-качество, разрешение, fps | перекодирование |
| 3 | **Без перекодирования** | смена контейнера (→ mp4/mkv); извлечение дорожек (аудио, субтитры); удаление лишних дорожек | мгновенно, `-c copy` |
| 4 | **Резка и склейка** | обрезка по таймкодам; разрезка на равные куски; склейка аудио (склейка видео — фаза 2) | copy или перекодирование |
| 5 | **Инспекция** | отчёт ffprobe в человеческом виде; кнопки «Открыть в ffplay», «Скопировать отчёт», сырой JSON | чтение |
| 6 | **Инструменты** | кадр в PNG; GIF из отрезка; скорость ×0.5–×2 (atempo); поворот видео; правка метаданных | разное |

### 2.3. Единый каркас вкладки

Все вкладки (кроме «Инспекции») построены по одной схеме, сверху вниз:

1. **Список файлов** — многострочный, с колонкой статуса. Кнопки
   «Добавить...», «Удалить выбранное», «Очистить». Добавление —
   перетаскиванием мышью в любое место вкладки или кнопкой.
   Строки можно перетаскивать для изменения порядка — файлы
   обрабатываются в порядке списка.
2. **Настройки операции** — пресеты + раскрывающийся блок «Расширенно».
   Панель настроек ни к какому файлу не привязана — это «заготовка
   следующего задания», её можно крутить в любой момент, в том числе
   пока очередь работает.
3. **Строка вывода** — шаблон имени результата, папка вывода
   (по умолчанию «рядом с исходником»), флажок «Перезаписывать готовые»
   (выключен — файлы с уже существующим результатом пропускаются,
   удобно перезапускать прерванную очередь).
4. **Кнопки управления:**
   - **«В очередь»** — контекстная: для файла «ожидает» ставит задание
     (снимок текущих настроек); для файла «в очереди» превращается в
     **«Отменить задание»**; для файла «в работе» неактивна.
   - **«В очередь: все»** — ставит задания всем ожидающим файлам списка;
     файлы в очереди/в работе/готовые пропускаются молча.
   - **«Старт»** — запускает обработку очереди.
   - **«Стоп»** со стрелочкой ▾ (split-button):
     основной клик — немедленный стоп (гасит текущий ffmpeg, очередь на
     паузу); в меню: **«Пропустить текущее задание»** (текущий файл
     помечается `⏭ пропущено`, очередь продолжается) и **«Пауза после
     текущего задания»** (мягкая пауза; работает как галочка — повторный
     выбор отменяет; состояние видно в статусной полосе).
   - «Открыть лог», «Очистить консоль», счётчик времени
     (общее | текущий файл).
5. **Консоль** — лог с прогрессом в процентах; флажок «Подробный вывод
   ffmpeg» в расширенном режиме. Перед стартом каждого файла в консоль
   печатается полная команда ffmpeg (`>>> ffmpeg -i ...`) — для
   прозрачности, отладки и связки с полем «Доп. параметры ffmpeg».

«Инспекция» отличается: один файл, вместо консоли — отчёт.

### 2.4. Очередь заданий

Отдельной вкладки/окна очереди нет — очередь видна как статусы файлов
прямо в списках. Жизненный цикл файла:

```
ожидает → ⏳ в очереди → ► в работе → ✓ готово / ✗ ошибка / ⏭ пропущено
```

- **Один файл — одно задание.** Задание = снимок настроек в момент
  постановки; последующее кручение ползунков на него не влияет.
  Редактирования заданий нет: «Отменить задание» → поправить настройки →
  поставить заново.
- В статусной колонке — краткая сводка задания:
  `⏳ в очереди · opus 64k, моно, норм.`; полный список настроек — во
  всплывающей подсказке строки.
- Готовый файл можно поставить в очередь снова (например, теперь в mp3).
- **Порядок выполнения** — порядок списка (внутри вкладки); между
  вкладками — в порядке появления первого задания на вкладке.
  Одновременно работает один ffmpeg.
- **«Старт» при пустой очереди**: диалог «Поставить в очередь все файлы
  (N шт.) с текущими настройками и начать?» с галочкой «Больше не
  спрашивать» (сохраняется в настройках).
- Ошибка файла не останавливает очередь: файл помечается `✗`, очередь
  идёт дальше, в конце — итог «готово: N, ошибок: M, пропущено: K».
- Недописанный файл-результат при ошибке/остановке/пропуске удаляется —
  битых огрызков не остаётся.
- Продолжение после любой паузы — кнопкой «Старт».

### 2.5. Защита от глупостей

- Результат совпал бы с исходником по пути — задание пропускается с
  предупреждением, исходник не затирается.
- Один и тот же файл нельзя добавить в список дважды; не-медиа файлы
  отсеиваются при добавлении (проверка ffprobe) с пометкой в консоли.
- Перед стартом — грубая проверка свободного места в папке вывода.
- **Проверка при старте программы:** ffmpeg/ffprobe на месте и
  запускаются; заодно читается список кодировщиков (`ffmpeg -encoders`).
  Нет ffmpeg — внятное сообщение сразу; нет кодировщика (урезанная
  сборка) — соответствующий формат показан серым с подсказкой
  «нет в вашей сборке ffmpeg», а не ошибка на середине очереди.

## 3. Вкладка «Аудио»

### 3.1. Назначение

Сжатие и конвертация звука из любых аудио/видео файлов (из видео берётся
звуковая дорожка; видеоряд отбрасывается). Наследник вкладки «Файл для
прослушивания» Transcriber XXL — вся её логика переносится.

### 3.2. Форматы и пресеты

Формат: **opus / mp3 / aac / flac / wav**.

Пресеты качества (для сжимающих форматов):

| Пресет | opus | mp3 | aac |
|---|---|---|---|
| Компактный — голос | 32k | 64k | 48k |
| Стандарт — голос | 64k | 128k | 96k |
| Высокое — голос | 96k | 192k | 128k |
| Максимум — музыка | 256k | 320k | 256k |
| Свой | значения из расширенного режима | | |

Для flac/wav пресетов качества нет (форматы без потерь) — блок пресетов
заменяется их настройками из расширенного режима (п. 3.3).

### 3.3. Расширенный режим (раскрывающийся блок)

Набор настроек зависит от выбранного формата:

- **opus**: битрейт 16–320k (ползунок); режим «Авто — кодек сам
  распознаёт речь/музыку» (умолчание, `application=audio`) / «Речь»
  (принудительный уклон, `voip`); VBR вкл/выкл.
- **mp3**: переключатель CBR/VBR; CBR — битрейт 32–320k,
  VBR — качество V0–V9.
- **aac**: переключатель CBR/VBR; CBR — битрейт 32–320k,
  VBR — качество 1–5. Профиль — только LC (встроенный кодировщик
  ffmpeg другого не умеет; HE-AAC сознательно не поддерживается —
  его нишу низких битрейтов закрывает opus).
- **flac**: уровень сжатия 0–8 (по умолчанию 5); разрядность
  «как в исходнике»/16/24 бит. Подсказка обязана объяснить: уровень
  сжатия НЕ влияет на качество (формат без потерь) — только на размер
  файла и время кодирования.
- **wav**: разрядность «как в исходнике» (умолчание, берётся из
  ffprobe) / 16 бит (совместимость) / 24 бит / 32 бит float
  (для дальнейшей обработки).
- **Общие для всех форматов**: частота дискретизации
  (авто/48к/44.1к/24к/16к).
- Изменение любой настройки переключает пресет на «Свой».

### 3.4. Дорожки и каналы

Тот же виджет списка дорожек, что на вкладке «Без перекодирования»
(п. 5.2): галки с реальными данными из ffprobe — кодек, каналы,
битрейт и имя дорожки, если прописано в файле (OBS пишет имена).

```
Дорожки:  ☑ №1 aac стерео 160k  (Track1 — микс)
          ☐ №2 aac моно 96k     (Track2 — микрофон)
          ☐ №3 aac стерео 160k  (Track3 — системные звуки)
Каналы:   [Как в исходнике ▾]
```

- У файлов с одной аудиодорожкой блок дорожек **прячется целиком**
  (нечего выбирать — не пугаем новичка); появляется только для
  мультитрека.
- Смысл галок зависит от режима каналов:
  - **«Как в исходнике»** — кодируются только отмеченные дорожки.
    Одна галка → один файл (типовой сценарий: из записи стрима —
    только дорожка микрофона в компактный opus). Несколько галок →
    каждая дорожка отдельным файлом с тегом `{дорожка}` в имени
    (пометка в консоли) — раскладка мультитрека на файлы.
  - **«Моно (сведение)»** — отмеченные дорожки сводятся в одну.
  - **«Стерео: голоса по ушам»** — галки заменяются парой комбобоксов
    «Левое ухо / Правое ухо» со списком реальных дорожек файла.
- Пакетная логика — как в п. 5.4: галки по первому файлу, к остальным
  применяются по номерам.
- Нормализация громкости: EBU R128 loudnorm, двухпроходная; в
  мультитреке каждая дорожка нормализуется отдельно до сведения.
  При выборе пресета «Максимум — музыка» нормализация снимается
  автоматически (голосовая цель −16 LUFS музыке вредит).
- Галка нормализации и целевая громкость живут парой в основной части
  вкладки: `☑ Нормализация громкости  Цель: [−16 LUFS ▾]`.
  Выпадайка активна только при включённой галке; варианты с
  подсказками: −14 (громче, стриминг) / −16 (речь, умолчание) /
  −18 (тише, аудиокниги) / −23 (вещательный стандарт EBU).

### 3.5. Шаблоны имён результата

Система тегов транскрибера, каждый тег — с английским синонимом
(работают оба всегда, независимо от языка интерфейса):

| Русский | Английский | Значение |
|---|---|---|
| `{имя}` | `{name}` | имя исходника без расширения |
| `{дата}` | `{date}` | дата исходного файла, ГГГГ-ММ-ДД |
| `{время_чм}` | `{time_hm}` | время исходника, ЧЧ-ММ |
| `{время_чмс}` | `{time_hms}` | время исходника, ЧЧ-ММ-СС |
| `{сегодня}` | `{today}` | сегодняшняя дата |
| `{#}` `{##}` `{###}` | — | номер файла в пачке (7 / 07 / 007) |

### 3.6. Умолчания

opus · «Стандарт — голос» · каналы «как в исходнике» · нормализация
вкл · шаблон `{имя}` · вывод рядом с исходником. Исходники никогда не
удаляются (принцип 1.3.5) — опции «удалить исходник» в программе нет.

### 3.7. Доп. параметры ffmpeg

В расширенном режиме — поле «Доп. параметры ffmpeg» (как в
транскрибере): содержимое добавляется в команду после параметров
программы, поэтому перекрывает их (у ffmpeg побеждает последняя
опция). Поле есть на всех перекодирующих вкладках (Аудио, Видео,
Резка и склейка). Ответственность за содержимое — на пользователе;
ошибки ffmpeg показываются в консоли по конкретному файлу, очередь
продолжается.

## 4. Вкладка «Видео»

### 4.1. Назначение

Сжатие видео: «файл не влезает в письмо/мессенджер/на диск».
Видеоряд перекодируется; звук по умолчанию копируется как есть —
это быстрее и бережёт качество (звук и так обычно мал).

### 4.2. Кодеки и пресеты

Кодек: **H.264** (умолчание — играет везде) / **H.265** (файл меньше
при том же качестве, но кодируется медленнее и поддерживается не
всеми устройствами).

Пресеты — по цели, а не по цифрам:

| Пресет | Смысл | Под капотом |
|---|---|---|
| Компактный | максимальное сжатие, для пересылки | CRF 28, preset slow |
| Стандарт | баланс размер/качество | CRF 23, preset medium |
| Качество | почти неотличимо от исходника | CRF 18, preset slow |
| Свой | значения из расширенного режима | — |

### 4.3. Разрешение и частота кадров

- Разрешение: как в исходнике / 1080p / 720p / 480p. Только
  уменьшение: если исходник меньше выбранного — файл не растягивается,
  разрешение остаётся исходным (пометка в консоли).
- Частота кадров: как в исходнике / 30 / 24.

### 4.4. Звук

Копировать как есть (умолчание) / перекодировать в aac с выбором
битрейта / удалить (немое видео).

### 4.5. Расширенный режим

- **CRF-ползунок 16–32** с живой подсказкой под ним
  («18 — визуально без потерь · 23 — стандарт · 28 — заметное сжатие»);
- **скорость кодирования** (preset): fast / medium / slow
  (подсказка: «медленнее = меньше файл при том же качестве»);
- **Доп. параметры ffmpeg** (п. 3.7). Короткий тултип с примерами
  команд + кнопка **«?»** рядом с полем — открывает немодальное окно
  «Справка: аппаратное кодирование» с прокруткой и моноширинным
  шрифтом для команд (можно держать открытым и копировать).
  Содержимое окна — п. 4.5.1.

#### 4.5.1. Текст справки «Аппаратное кодирование»

**Общее**

- Аппаратный кодировщик (чип видеокарты) работает **в 5–15 раз
  быстрее** программного, но файл при том же качестве получается
  на **20–40% больше**. Для архива навсегда — лучше программный;
  для «быстро пережать и отправить» — аппаратный.
- Ваши параметры перекрывают настройки программы (в команде ffmpeg
  действует последняя опция).
- **Проверка, что ускорение работает**, без траты времени: поставьте
  в очередь короткий ролик. Ошибка «Cannot load nvcuda.dll» — нет
  карты NVIDIA или очень старый драйвер; «No device available» у
  QSV — встройка выключена в BIOS или нет драйвера.
- **Размер вместо качества.** У любого кодировщика (аппаратного и
  программного) есть два способа управления: по качеству
  (`-cq` / `-global_quality` / `-qp`) — качество держится, размер
  непредсказуем; или по битрейту — вместо параметра качества
  укажите `-b:v 4M` (целевые 4 Мбит/с), тогда размер предсказуем
  (≈ битрейт × длительность), а качество подстраивается. Удобно,
  когда файл должен влезть в лимит. Пример:
  `-c:v h264_nvenc -preset p5 -b:v 4M`.

**NVIDIA (NVENC)**

```
-c:v h264_nvenc -preset p5 -cq 27
-c:v hevc_nvenc -preset p5 -cq 29
-c:v h264_nvenc -preset p5 -b:v 4M   (битрейт вручную)
```

- `-c:v h264_nvenc` — кодировать видео чипом NVENC карты NVIDIA
  (h264 — максимальная совместимость; `hevc_nvenc` — H.265, файл
  меньше).
- `-preset p1`…`p7` — баланс скорость/качество: p1 — быстрейший и
  худший, p7 — медленнейший и лучший. Разумно **p5–p6** (даже p7 у
  NVENC в разы быстрее программного кодирования).
- `-cq 0–51` — постоянное качество, **меньше = лучше = крупнее
  файл**. Ориентиры: 23 — высокое, 27 — стандарт, 32 — сильное
  сжатие; разумные пределы 19–35. Шкала жёстче программного CRF —
  прибавляйте к привычному CRF 3–5 (CRF 23 ≈ CQ 27).
- Требования: GeForce GTX 600+ (для HEVC — GTX 950+); свежий обычный
  драйвер GeForce (NVENC встроен в него). BIOS не трогается.

**Intel (Quick Sync / QSV)**

```
-c:v h264_qsv -global_quality 25
-c:v hevc_qsv -global_quality 27
-c:v h264_qsv -b:v 4M                (битрейт вручную)
```

- `-global_quality 1–51` — качество, **меньше = лучше**. Ориентиры:
  22 — высокое, 25 — стандарт, 30 — сильное сжатие; пределы 18–35.
- Требования: процессор Intel со встроенной графикой + драйвер Intel
  Graphics. Засада: на десктопах с дискретной картой встройка часто
  **выключена в BIOS** («Integrated Graphics» / «iGPU Multi-Monitor»
  → Enabled); монитор к ней подключать не обязательно. На
  процессорах серии F (i5-12400F и т.п.) встройки нет — QSV
  невозможен.

**AMD (AMF)**

```
-c:v h264_amf -quality quality -rc cqp -qp_i 22 -qp_p 24
-c:v h264_amf -quality quality -rc cbr -b:v 4M   (битрейт вручную)
```

- `-quality speed | balanced | quality` — пресет скорости; ставьте
  `quality`, он всё равно быстрый.
- `-rc cqp` — режим постоянного качества; `-qp_i` / `-qp_p` 0–51 —
  качество I- и P-кадров, **меньше = лучше**, разумно 20–30
  (P на 2 больше I).
- Требования: Radeon от ~2013 года (GCN и новее) + драйвер
  Adrenalin. BIOS не трогается.

### 4.6. Режим «уложиться в размер»

Третий способ управления качеством (переключатель в блоке качества):

```
Качество: (•) Пресет [Стандарт ▾]
          ( ) CRF (значение — в расширенном режиме)
          ( ) Размер файла: [ 25 ] МБ
```

Битрейт видео = (размер × 8 / длительность) − битрейт аудио − ~3% на
служебные данные контейнера. Кодирование **двухпроходное**
(`-pass 1/2`): первый проход строит «карту сложности», второй кодирует
по ней; попадание в размер 1–2%, цена — время ×1.5–2.

Все проверки — в момент постановки в очередь (данные ffprobe уже в
кэше с момента добавления файла):

1. **«Слишком мало»**: посчитанный битрейт сравнивается с порогами
   «будет каша» по разрешению исходника (~250 kbps для 480p, ~500 для
   720p, ~1000 для 1080p). Ниже порога — диалог с вычисленной
   конкретикой и тремя кнопками: **[Понизить до NNNp и уложиться]**
   (рекомендация: программа сама подбирает разрешение, при котором
   битрейт становится нормальным) / [Кодировать как есть] / [Отмена].
2. **«Слишком много»**: если битрейт выше потолка разумного
   (для 1080p H.264 ~8 Мбит/с) — задание молча переводится в режим
   качества (CRF 18), пометка в консоли: «запрошенный размер больше
   необходимого — кодирую по качеству, выйдет меньше лимита».
3. **Пачка**: лимит — для каждого файла по отдельности (у каждого
   своя длительность → свой битрейт). Зафиксировано в подсказке поля.
4. **Звук**: при включении режима звук автоматически переключается на
   «aac 128k» (пометка в консоли). Если пользователь вернул
   «копировать» и звук занимает >30% бюджета размера — предупреждение
   с цифрами и предложением aac 96k.
5. **Контроль результата**: после кодирования в консоль — факт:
   «Готово: 24.1 МБ из 25 (попадание 96%)».

### 4.7. Аппаратное ускорение — по фазам

- **Фаза 1**: только программное кодирование (libx264/libx265) — лучшее
  сжатие, работает у всех одинаково. Настройка «Кодировщик» существует
  в интерфейсе с первого дня с единственным пунктом «Программный» —
  гнездо под фазу 2. Экспертный путь к NVENC/QSV/AMF уже открыт через
  поле доп. параметров (п. 4.5).
- **Фаза 2**: NVENC как честный пункт «Кодировщик»: маппинг CRF→CQ и
  пресетов, пробный запуск при старте, внятные сообщения об
  недоступности (нет карты/старый драйвер).
- **QSV/AMF**: только по живому запросу и при наличии машины для
  тестирования.

### 4.8. Умолчания

H.264 · «Стандарт» · разрешение и fps как в исходнике · звук
копировать · шаблон `{имя}` · вывод рядом с исходником.

## 5. Вкладка «Без перекодирования»

### 5.1. Назначение

Операции, где данные не перекодируются, а копируются (`-c copy`):
мгновенно и без потери качества даже на часовых файлах. Два режима:

```
(•) Пересобрать файл          ( ) Извлечь дорожки
```

### 5.2. Режим «Пересобрать файл»

Смена контейнера и/или отбор дорожек — за один проход.

```
Контейнер: [mp4 ▾]        ← mp4 / mkv / mka (только звук) / как в исходнике
Дорожки:   ☑ Видео h264 1080p
           ☑ Аудио №1 aac rus
           ☐ Аудио №2 ac3 eng
           ☐ Аудио №3 ac3 jpn
           ☐ Субтитры №1 srt eng
           [Все] [Без звука]
```

- **Список дорожек** строится из ffprobe (кодек, разрешение/частота,
  язык). Галка = оставить; по умолчанию отмечены все. Кнопки быстрого
  выбора: «Все», «Без звука» (снимает все аудио-галки).
- Типовые сценарии: сменить контейнер (галки не трогать); выкинуть
  лишние языки из 10-дорожечного mkv (снять галки); и то и другое
  сразу.
- **Контейнеры:**
  - **mp4** (умолчание) — всегда `-movflags +faststart` (мгновенный
    старт воспроизведения при пересылке/стриминге);
  - **mkv** — всеяден, всегда чистое копирование;
  - **mka (только звук)** — Matroska без видео: галка «Видео» гаснет
    с подсказкой «mka — звуковой контейнер, видеоряд отбрасывается»;
    все отмеченные аудиодорожки копируются как есть (Matroska
    принимает любые кодеки), главы и метаданные сохраняются. Ценность
    против «Извлечь дорожки»: несколько дорожек (например, все языки
    фильма) в одном компактном файле без потерь. Тултип-честность:
    «mka играют VLC, foobar2000, MPC; телефоны, телевизоры и
    автомагнитолы — обычно нет. Формат для хранения, не для
    совместимости»;
  - **как в исходнике** — контейнер не меняется, только отбор дорожек.
- **Аудио при выходе в mp4** (mkv/mka всеядны — для них настройка не
  участвует). Несовместимое аудио (DTS, TrueHD, FLAC...)
  автоматически перекодируется; в консоли — предупреждение с
  подсказкой, где ручка: `⚠ дорожка 2 (DTS 5.1) несовместима с mp4 →
  перекодирована в aac 192k (изменить — в блоке «Расширенно»)`.
  Блок «▸ Расширенно»:

  ```
  Аудио при смене контейнера:
    Кодек: [aac ▾]   Битрейт: [192k ▾]
    ☐ Перекодировать всё аудио (не только несовместимое)
  ```

  - Кодеки: **aac** (умолчание; 96/128/**192**/256/320/512k) /
    **ac3** (только табличные значения стандарта:
    192/224/256/320/384/**448**/512/576/640k) / **opus**
    (64/96/**128**/160/192/256k) / «пропустить дорожку».
    Комбобокс битрейта перестраивается под кодек (у ac3 произвольных
    битрейтов не бывает — потому списки, а не ползунок).
  - Тултип выбора кодека: «aac — играет везде · ac3 — старые
    ресиверы/кинотеатры, 5.1 · opus — лучшее сжатие, но в mp4 его
    понимают не все плееры и телевизоры: ставьте, только если уверены
    в своём устройстве».
  - Галка снята (умолчание): совместимые дорожки копируются без
    потерь, настройка касается только несовместимых. Галка стоит:
    в выбранный кодек/битрейт перегоняются все аудиодорожки
    (в консоли: `дорожка 1 (aac) → ac3 448k (перекодировать всё:
    вкл)`).
  - Раскладка каналов сохраняется (5.1 остаётся 5.1).
- **Субтитры**: в mp4 — текстовые конвертируются в mov_text,
  картиночные (PGS) отбрасываются с пометкой в консоли; в mkv/mka —
  копируются как есть.
- **Защиты**: нельзя снять все галки (пустой файл не собирается);
  «без видео, но с аудио» — законно (звук в контейнере).
- Никаких диалогов посреди очереди: правила заданы заранее, консоль
  фиксирует принятые решения по каждому файлу.

### 5.3. Режим «Извлечь дорожки»

- Тот же список дорожек с галками, но каждая отмеченная дорожка
  сохраняется **отдельным файлом** в родном формате; расширение
  программа выбирает сама по кодеку (aac → .m4a, opus → .opus,
  mp3 → .mp3, pcm → .wav, flac → .flac, h264/h265 → .mp4,
  srt → .srt).
- Шаблон имени получает тег `{дорожка}` / `{track}` (номер и тип).
- Для одной дорожки удобен этот режим; для «несколько дорожек одним
  файлом» — mka в режиме «Пересобрать» (п. 5.2).

### 5.4. Пакетная работа с дорожками

Списки дорожек у файлов разные. Галки задаются по **первому
выбранному** файлу, к остальным применяются «по номерам и типам»
(аудио №1, субтитры №2...); нет такой дорожки — пропуск с пометкой в
консоли. Типовой случай «сезон сериала одинаковой структуры»
покрывается идеально.

## 6. Вкладка «Резка и склейка»

### 6.1. Режимы

```
(•) Обрезать   ( ) Разрезать на части   ( ) Склеить
```

Работа формами и пачками, без таймлайна и предпросмотра (принцип
1.3.3): границы задаются таймкодами — например, из транскрипта.
Для визуальной резки «глазами» — Machete (рекомендация в README).

### 6.2. Режим «Обрезать»

```
Фрагменты: [ 5:20-12:40, 31:00-35:15                        ]  (?)

Действие:  (•) Оставить эти куски    ( ) Вырезать эти куски
Результат: (•) Отдельными файлами    ( ) Одним файлом

Рез: (•) Мгновенный, без перекодирования   ( ) Точный (с перекодированием)
```

- **Фрагменты** — одной строкой, через запятую. Форматы таймкодов
  свободные: `5:20`, `05:20`, `1:05:20`, `5:20.5`. Пустой край =
  от начала / до конца: `-5:20` (первые 5:20), `31:00-` (с 31-й
  минуты до конца). Кнопка `(?)` — подсказка с примерами.
- **Действие**: «Оставить» — на выходе указанные куски; «Вырезать» —
  программа инвертирует интервалы по длительности из ffprobe
  (указали `31:00-35:15` → остаётся `0–31:00` и `35:15–конец`).
- **Результат**: «Отдельными файлами» — каждый кусок в свой файл,
  в шаблоне имени тег `{фрагмент}` / `{part}`; «Одним файлом» —
  куски склеиваются подряд, тег не используется.
- Все четыре комбинации осмысленны:

| Действие | Результат | Сценарий |
|---|---|---|
| Оставить | Отдельными | нарезать цитаты из лекции |
| Оставить | Одним | компиляция лучших моментов |
| Вырезать | Одним | убрать рекламу/паузы/кашель |
| Вырезать | Отдельными | развалить запись по разделителям |

- **Рез**:
  - «Мгновенный» — `-c copy`; для видео честная подпись: «граница
    уедет к ближайшему ключевому кадру (±1–5 с)»; при «Одним файлом»
    каждый стык — два таких реза (подпись предупреждает); для аудио
    рез точный всегда.
  - «Точный» — с перекодированием: разворачивается мини-блок
    качества по типу файла (аудио → формат/пресет вкладки «Аудио»,
    видео → кодек/пресет вкладки «Видео»); стыки чистые.

### 6.3. Режим «Разрезать на части»

```
Длина части: [ 10 ] мин
```

- Без перекодирования (`-f segment`). Для видео куски встают по
  ключевым кадрам (реально 10:02, 9:57 — подпись предупреждает),
  для аудио — ровно.
- Имена: шаблон `{имя} {##}` → `запись 01.mp3`, `запись 02.mp3`...

### 6.4. Режим «Склеить»

Смысл списка файлов меняется, интерфейс это показывает: заголовок
списка — **«Порядок склейки (перетаскивайте строки)»**, строки
нумеруются, всё задание — одно на весь список (кнопка «В очередь:
все» прячется, остаётся «В очередь»).

**Живая строка-вердикт** — сразу после добавления файлов программа
сверяет их параметры (ffprobe) и до постановки в очередь показывает,
каким путём пойдёт склейка:

- **Аудиофайлы, параметры совпадают** (кодек, частота, каналы):
  `✓ параметры совпадают → склейка мгновенная, без перекодирования`
  (concat demuxer, ноль потерь).
- **Аудиофайлы, параметры различаются**:
  `параметры различаются (m4a 44.1кГц стерео + mp3 48кГц моно) →
  склейка через перекодирование в: [opus ▾] [Стандарт — голос ▾]` —
  разворачивается мини-блок формата (умолчание opus «Стандарт»).
  Общий знаменатель: частота — наибольшая из встреченных; каналы —
  стерео, если хоть один файл стерео, иначе моно. Всё одной командой
  ffmpeg (фильтр concat), без промежуточных файлов.
- **Галка «☑ Выровнять громкость кусков»** — каждый кусок проходит
  loudnorm (п. 3.4) до склейки; лечит «куски записаны с разной
  громкостью, стык режет ухо». По умолчанию включена на пути
  перекодирования. На пути «без перекодирования» её включение
  переводит склейку в перекодирование — вердикт сразу предупреждает
  (выровнять громкость без перекодирования нельзя в принципе).
- **Видеофайлы — фаза 1: только без перекодирования.** Параметры
  совпадают (типично: куски одной записи) → мгновенная склейка.
  Различаются → отказ с сообщением:
  «К сожалению, склейку этих файлов без перекодирования выполнить
  невозможно: 1920×1080 h264 + 1280×720 hevc. Склейка с
  перекодированием, возможно, появится в будущих версиях. Пока —
  сторонний видеоредактор: DaVinci Resolve (бесплатный), MAGIX Vegas
  Pro, Adobe Premiere Pro, AVS Video Editor.»
- Смешанный список (аудио вперемешку с видео) → отказ с пояснением.

**Фаза 2 — склейка видео с перекодированием** (решения зафиксированы
заранее): оба ролика приводятся к общему знаменателю и склеиваются
за один проход. Целевые параметры по умолчанию — от «лучшего» файла
(наибольшее разрешение; H.264 для совместимости), редактируются в
мини-блоке. Разные пропорции — меньшее вписывается с чёрными полями
(letterbox), с предупреждением в вердикте; разный fps — приводится к
большему.

### 6.5. Умолчания

Режим «Обрезать» · действие «Оставить» · результат «Отдельными
файлами» · рез «Мгновенный». В «▸ Расширенно» — доп. параметры
ffmpeg (п. 3.7) и подробный вывод.

## 7. Вкладка «Инспекция»

- **Один файл** (перетащили или выбрали — без списка и очереди;
  анализ мгновенный, ffprobe).
- Сверху — **человеческая сводка**: имя, контейнер, длительность,
  размер, суммарный битрейт.
- Ниже — **таблица дорожек**: тип (видео/аудио/субтитры), кодек,
  детали (разрешение+fps / частота+каналы+битрейт), язык, имя
  дорожки, оценка размера.
- Кнопки: **[Открыть в ffplay]** (быстро глянуть/послушать;
  закрывается Esc), **[Скопировать отчёт]** (текстом в буфер),
  **[▸ Сырой JSON]** (раскрывающийся блок с полным выводом ffprobe).
- **Строка-подсказка**: программа комментирует файл с точки зрения
  других вкладок — «10 аудиодорожек — можно проредить на вкладке
  “Без перекодирования”», «звук ac3 448k — на вкладке “Аудио”
  сожмётся в opus ~в 7 раз без слышимой потери». Дёшево, а новичка
  учит возможностям программы.

## 8. Вкладка «Инструменты»

Пять мини-режимов переключателем; каркас (список файлов, очередь,
консоль) — общий, как везде:

```
(•) Кадр из видео  ( ) GIF  ( ) Скорость  ( ) Поворот  ( ) Метаданные
```

### 8.1. Кадр из видео

Три способа, переключателем:

1. **По таймкодам** — одной строкой через запятую
   (`5:20, 12:33, 1:05:00`), каждый таймкод → отдельная картинка.
2. **N кадров равномерно** — обзорная нарезка по всей длине
   (найти нужное место в длинном файле).
3. **Все кадры из диапазона** — «охота за идеальным кадром»:
   поле «с — по» (умолчание — 3 секунды от указанного места), под ним
   **живой счётчик** из fps файла: `≈ 90 кадров, ~12 МБ в JPG /
   ~270 МБ в PNG`. Мягкий предохранитель на **300 кадрах**: больше —
   диалог «Получится 1450 кадров и ~2 ГБ. Точно?» (от опечатки в
   таймкоде). Формат по умолчанию в этом способе — JPG.

Формат: **PNG** (без потерь — слайды, текст) / **JPG + ползунок
качества 60–100%** (умолчание 90; маппится на `-q:v`) — в разы меньше
для «кадр как фотка». Имена: `{имя} {таймкод}.png/jpg`, для диапазона
— `{имя} {таймкод}{###}.jpg`.

### 8.2. GIF из отрезка

```
Отрезок: [5:20-5:28]   Ширина: [480 ▾]   fps: [15 ▾]
Пресет: [Стандарт ▾]   Формат: [GIF ▾ / WebP]        (?)
```

- Всегда двухпроходно с оптимальной палитрой (palettegen →
  paletteuse).
- Пресеты:

| Пресет | Внутри |
|---|---|
| Компакт | 128 цветов, дизеринг bayer, только изменившиеся области |
| Стандарт | 256 цветов, bayer, только изменившиеся области |
| Максимум | 256 цветов, floyd_steinberg, палитра с приоритетом движения |

- **▸ Расширенно** (трогаешь — пресет на «Свой»): цвета (32–256),
  дизеринг (нет / bayer / floyd_steinberg), приоритет движущихся
  областей (галка), только изменившиеся области (галка),
  зацикливание (бесконечно / 1 раз / N), скорость (×1–×2).
- **Формат WebP** (анимированный): в 2–4 раза меньше GIF при том же
  качестве; тултип-честность: «не все мессенджеры и форумы понимают
  его как анимацию».
- Подпись-предупреждение: «GIF — прожорливый формат: 10 секунд 480px
  ≈ 3–8 МБ; для пересылки видео лучше mp4 со вкладки “Видео”».
- После кодирования — факт в консоль: `GIF готов: 4.2 МБ (480px,
  15fps, 8.2 с)`.
- Кнопка **«?»** — окно «Справка: GIF» с прокруткой (как справка по
  аппаратному кодированию, п. 4.5.1). Содержимое — п. 8.2.1.

#### 8.2.1. Текст справки «GIF»

**Почему GIF такой толстый.** Формату 35+ лет: максимум 256 цветов
на кадр и нет настоящего межкадрового сжатия, как в видео. Поэтому
10 секунд GIF могут весить больше минуты mp4. GIF берут за
автовоспроизведение везде — но если получатель переживёт mp4,
шлите mp4.

**Рычаги размера, по силе действия:**

1. **Ширина** — размер растёт примерно квадратично: 480 → 360px
   почти вдвое легче. Для мессенджера часто хватает 360.
2. **fps** — 10–15 достаточно для большинства сцен; 20–25 — только
   для плавного быстрого движения (спорт, игры).
3. **Длительность** — лучшие гифки короче 10 секунд. Длиннее 15 —
   почти всегда повод для mp4.
4. **Цвета** — 256 → 128 экономит 20–40%; на скринкастах,
   мультфильмах, интерфейсах разницы не видно. Живое видео с
   градациями кожи/неба — оставляйте 256.
5. **Дизеринг** — способ маскировать нехватку цветов:
   - *bayer* — лёгкий узор-сеточка, компромисс размер/вид
     (умолчание);
   - *floyd_steinberg* — «шумовое» размытие: градиенты заметно
     красивее, файл на 30–50% тяжелее;
   - *нет* — минимальный размер, но на небе и коже — полосы.

**Тонкие настройки:**

- *Приоритет движущихся областей* — палитра строится точнее там, где
  движение (для живых сцен; статичному скринкасту не нужно).
- *Только изменившиеся области* — перерисовывается лишь изменившийся
  прямоугольник кадра: скринкасты и «говорящая голова на статичном
  фоне» худеют в разы без потери качества. Выключать почти незачем.
- *Зацикливание* — бесконечно (умолчание) / один раз / N раз.
- *Скорость* — ×1.5–×2 уплотняет длинный фрагмент в короткую гифку.

**WebP вместо GIF** — тот же отрезок в 2–4 раза меньше и с полными
цветами. Понимают браузеры и большинство мессенджеров; не понимают
старые форумы и некоторые приложения (покажут статичную картинку).
Если аудитория — браузер/телефон, WebP почти всегда лучше.

### 8.3. Скорость

- Пресеты ×0.5 / ×0.75 / ×1.25 / ×1.5 / ×2 + точное поле ввода.
- Звук через atempo — **тон сохраняется** (лектор не превращается в
  бурундука); видео подгоняется автоматически.
- Работает и для чистого аудио (ускорить подкаст/аудиокнигу).

### 8.4. Поворот

- 90° по часовой / 90° против / 180° (видео с телефона, снятое
  боком).
- Переключатель: **быстрый** — правится только флаг поворота в
  метаданных (мгновенно, без потерь; честная подпись: «старые плееры
  флаг игнорируют») / **надёжный** — перекодирование с настройками
  качества (кадры физически поворачиваются).

### 8.5. Метаданные

- Поля: название, исполнитель, альбом, год, жанр, комментарий;
  **обложка** для аудио (выбор картинки).
- Применяется через `-c copy` — мгновенно, звук/видео не
  перекодируются.
- Кнопка **«Очистить все метаданные»** (`-map_metadata -1`) — убрать
  хвосты перед публикацией.
- **Картинки (jpg/png/webp) тоже принимаются** — но только на
  очистку: вычищаются EXIF, GPS, серийник камеры, миниатюры — без
  перекодирования, пиксели не трогаются (сценарий «убрать геолокацию
  с фото перед отправкой»). Редактирование полей для картинок
  недоступно — ffmpeg не умеет корректно писать EXIF (честная
  подпись в интерфейсе; для этого есть exiftool).

## 9. Фазы разработки

- **Фаза 1 (MVP)**: каркас окна + темы + settings.json + очередь со
  статусами и статусной полосой; вкладки **«Аудио»** (перенос логики
  listening.py), **«Инспекция»**, **«Без перекодирования»**.
  Уже полезная программа.
- **Фаза 2**: **«Видео»** (включая режим «уложиться в размер»),

- **Фаза 3**: **«Инструменты»**, английский язык интерфейса.
- **Фаза 4 (по запросу)**: NVENC как пункт «Кодировщик» (п. 4.7),
  склейка видео с перекодированием (п. 6.4), кастомная отрисовка
  комбобоксов/скроллбаров под тему (п. 1.4).

## 11. Сборка и требования

- Сборка: `build.bat` → csc из .NET Framework 4.8
  (`C:\Windows\Microsoft.NET\Framework64\v4.0.30319\csc.exe`),
  без сторонних SDK. Иконка — `/win32icon:V.ico` (создание .ico —
  по отработанному конвейеру транскрибера: ffmpeg из PNG; помнить
  про кэш иконок Windows при пересборке).
- Требования: Windows 10/11 x64, .NET Framework 4.8 (предустановлен).
- Дистрибуция: папка целиком (exe + ffmpeg + docs), без установщика.
